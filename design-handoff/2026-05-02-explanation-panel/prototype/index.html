Explanation panel — right-rail inspector

Static spec for the four-tab (what / why / effect / impact) explanation panel that slides in from the right when a block is selected. Five result states, both themes, plus tab-persistence and resize affordance. Companion README.md covers intent, decisions, and open token questions.

  1. Header. Block label (Geist Sans 600, 14px) with role dot in the role-border colour. Sub-line is the block id and category in Geist Mono 11px. Mirrors the existing inspector header pattern.
  2. Close. Lowercase “close” in Geist Mono caps to match the existing inspector button. Keyboard-reachable, accent focus ring.
  3. Tabs. what / why / effect / impact, in that fixed order. The active tab is the only one with a 1.5px underbar in --fg; colour is never the only signal — the active tab is also aria-selected="true" and the only tab in the keyboard tab order.
  4. Body. Single 14px / 1.5 paragraph per the voice rules. effect and impact mix prose with shape callouts (3×4) and inline math placeholders ($$k = 4$$) — Geist Mono tabular nums for raw numbers.
  5. Value strip. Always-on glance at the current output regardless of active tab. Hidden in computing / error / unknown states. Mono, tabular, ellipsised.
  6. Resize handle. 6px hit area on the left edge, 1.5px visible affordance on hover/focus only. cursor: col-resize. Width range 320–520px clamped in JS.

Width

380px default

Resizable 320–520. Within the 360–420 range from the brief; persists per workspace, not per block.

Slide-in

220ms ease-out

Translate-X 16→0, opacity 0→1. Content fades 60ms behind. Disabled under prefers-reduced-motion.

Tab state

Workspace-persistent

Active tab survives selection changes (block A's "effect" stays "effect" on block B), unless the new block hides that tab — fall back to the leftmost available, matching explanation-tabs.tsx.

Contrast

AA · 4.5:1

Body text on --surface, both themes. Status chip text on tinted backgrounds checked separately — see tokens-used.md.

02

Result states · light

Five states required by the brief. Each shows panel chrome plus the tab-body content for that state. Frames in pairs so reviewers can scan diagonally.

03

Result states · dark

Same five states, dark theme. Tokens come from BRAND.md dark scale verbatim — no per-theme remix.

04

Tab persistence across selections

The active tab is workspace-scoped, not selection-scoped. If effect is open on block A and the user selects block B, they stay on effect.
